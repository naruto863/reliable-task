# ReliableTask v0.3.0

ReliableTask v0.3.0 is a preview release focused on reliability hardening, safer Admin defaults, real MySQL validation paths, and clearer production adoption guidance.

## Highlights

- Hardened worker lease handling so stale workers cannot overwrite newer claims or recovery state.
- Added explicit task state-machine validation and richer execution log trace fields.
- Added MySQL integration-test profiles for Testcontainers and dedicated local MySQL validation.
- Added safer Admin defaults: write APIs are disabled by default, with separate audit-log and batch-operation switches.
- Added explicit submission idempotency keys, custom retry strategy registration, failure classification, task events, and lower-cardinality metrics.

## What's Included

- Worker claim lock TTL configuration through `reliable-task.worker.lock-ttl-seconds`.
- Recovery based on expired `lock_expire_at` semantics with bounded reset scans.
- Handler-level concurrency enforcement through `TaskHandler.maxConcurrency()`.
- Retry/dead handling for executor lookup, submit rejection, timeout, and execution failures.
- `TaskSubmitRequest.idempotencyKey` for user-controlled submission deduplication keys.
- `RetryStrategyType.CUSTOM` resolution through registered retry strategies, plus jitter/min/max delay controls for exponential retry.
- `FailureClassifier`, `TaskEventListener`, and `TaskExceptionFormatter` extension points.
- Queue health metrics for backlog, oldest pending age, pending/running/dead totals, available worker capacity, and recovery counts.
- README and demo guidance for at-least-once semantics, handler idempotency, Admin security, and release validation.

## Compatibility Notes

- ReliableTask remains in the `0.x` preview stage. APIs, configuration, and database schema may still change before `v1.0.0`.
- The framework provides at-least-once execution, not exactly-once execution. Business handlers must remain idempotent.
- Admin read APIs are still registered by default. Admin write APIs are disabled by default and require `reliable-task.admin.write-enabled=true`.
- Reserved compatibility properties remain bindable but do not change runtime behavior: `reliable-task.serializer.type`, `reliable-task.store.table-prefix`, `reliable-task.admin.port`, and `reliable-task.admin.context-path`.
- ReliableTask is not published to Maven Central yet. Use a source build, local Maven installation, or a private Maven repository.

## Getting Started

Use the Spring Boot starter after building or installing from source:

```xml
<dependency>
    <groupId>com.reliabletask</groupId>
    <artifactId>reliable-task-spring-boot-starter</artifactId>
    <version>0.3.0</version>
</dependency>
```

See the README for MySQL schema initialization, minimal configuration, and demo instructions.
