# ReliableTask v0.5.0

ReliableTask v0.5.0 is a preview release focused on production operations readiness: safer Admin defaults, bounded troubleshooting queries, dead-letter extension points, monitoring guidance, and release-grade MySQL recovery validation.

## Highlights

- Added bounded read-only Admin operational queries for recent failures, slow executions, failure-top aggregation, and task lifecycle timeline views.
- Added `admin.query.*` guardrails for default windows, maximum windows, default limits, maximum limits, and slow-task thresholds.
- Added `TaskDeadLetterHandler` SPI, no-op default handler, dispatcher auto-configuration, and DEAD-path dispatch after successful state writes.
- Kept Admin REST APIs disabled by default, write APIs disabled by default, and authorization checks enabled by default when Admin is explicitly registered.
- Added production monitoring docs, Prometheus alert examples, incident runbook, production checklist, and release validation notes.

## What's Included

- Recent failed execution and slow execution Admin query APIs with bounded time windows and limits.
- Failure-top aggregation by `taskType`, `errorCode`, or `taskType,errorCode`.
- Lightweight task timeline built from task rows, execution logs, audit logs, and current task status.
- Dead-letter dispatch in no-handler and RetryEngine DEAD paths only after successful DEAD state writes.
- Documentation for payload sensitivity, Admin network isolation, authentication, authorization, audit, MySQL validation profiles, alert tuning, and recovery operations.
- README and demo guidance for v0.5 operational views, dead-letter SPI, production defaults, and local demo opt-in behavior.

## Compatibility Notes

- ReliableTask remains in the `0.x` preview stage. APIs, configuration, and database schema may still change before `v1.0.0`.
- The framework provides at-least-once execution, not exactly-once execution. Business handlers must remain idempotent.
- Admin REST APIs are disabled by default. If enabled, write APIs still require `reliable-task.admin.write-enabled=true`.
- `admin.query.*` defaults apply to the new v0.5 operational queries; existing `/tasks` and `/audit-logs` list APIs keep their compatible filtering behavior.
- Reserved compatibility properties remain bindable but do not change runtime behavior: `reliable-task.serializer.type`, `reliable-task.store.table-prefix`, `reliable-task.admin.port`, and `reliable-task.admin.context-path`.
- ReliableTask is not published to Maven Central yet. Use a source build, local Maven installation, or a private Maven repository.

## Getting Started

Use the Spring Boot starter after building or installing from source:

```xml
<dependency>
    <groupId>com.reliabletask</groupId>
    <artifactId>reliable-task-spring-boot-starter</artifactId>
    <version>0.5.0</version>
</dependency>
```

See the README for MySQL schema initialization, minimal configuration, Admin safety guidance, and demo instructions.
