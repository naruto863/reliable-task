# ReliableTask v0.7.0

ReliableTask v0.7.0 is a preview release focused on the standalone Web console. It keeps the core, executor, store, Worker starter, and Admin starter boundaries intact while adding an independent Vue/Vite operations UI for read-only troubleshooting and guarded Admin write operations.

## Highlights

- Added `reliable-task-console`, a standalone Vue 3, TypeScript, Vite, and Ant Design Vue console preview.
- Added console capabilities and console-safe task detail APIs so the UI can render payload-safe details without requesting raw payload by default.
- Added read-only troubleshooting views for dashboard stats, task list, task detail, execution logs, timeline, workers, stale workers, and audit logs.
- Added runbook-oriented Dashboard paths for backlog growth, dead task spikes, retry storms, and stale or missing workers.
- Added guarded single-task write UI for retry, cancel, requeue, and update payload.
- Added guarded batch preview, batch requeue, and batch cancel UI.
- Tightened backend Admin write guards so enabled writes still require auth, audit, and confirmation before mutation.

## What's Included

- Console-safe endpoint: `GET /api/reliable-task/console/tasks/{id}`.
- Capabilities endpoint: `GET /api/reliable-task/console/capabilities`.
- Payload preview defaults that keep plaintext disabled unless explicitly enabled.
- Frontend API client with `Result<T>` envelope handling, operator header injection, trace id creation, and confirmation header support for writes.
- Dashboard, Tasks, Task Detail, Workers, and Audit Logs views with loading, empty, forbidden, network, not-found, and disabled states.
- Single-task write confirmation requiring explicit `CONFIRM`.
- Batch operation preview requirement before execution, with bounded limit display.
- Local demo and Vite proxy documentation.
- Playwright smoke coverage for the read-only troubleshooting path using mocked Admin APIs.
- Readiness report: `docs/review/RELIABLE_TASK_V07_READINESS_REPORT.md`.

## Compatibility Notes

- ReliableTask remains in the `0.x` preview stage. APIs, configuration, and database schema may still change before `v1.0.0`.
- The console is independent from Maven and is not bundled into the Worker starter, Admin starter, or demo jar.
- Worker-only applications do not need the console and do not need Admin/Web dependencies.
- Console deployment requires an Admin-enabled Spring Boot application behind an internal operations boundary.
- Write buttons remain disabled unless backend capabilities report write, auth, audit, batch where applicable, and confirmation support.
- Payload plaintext reveal remains disabled by default. Do not rely on frontend hiding as the payload security boundary.
- Maven Central publication is still not performed.

## Getting Started

Start the demo backend:

```bash
mvn -pl reliable-task-demo -am spring-boot:run
```

Start the console:

```bash
cd reliable-task-console
npm install
npm run dev
```

Open `http://localhost:5173`. The Vite dev server proxies `/api/reliable-task` to `http://localhost:8080` by default.

For static deployment:

```bash
cd reliable-task-console
npm ci
npm run build
```

Deploy `dist/` behind the same protected operations boundary as the Admin API and proxy `/api/reliable-task` to the Admin-enabled application.
