# ReliableTask v0.6.0

ReliableTask v0.6.0 is a preview release focused on SPI and module-boundary stabilization. It keeps the v0.5 operational surface, narrows internal extension contracts, adds explicit payload/interceptor/trace/name extension points, separates Worker and Admin starters, and provides migration-tool entry points for initial schema setup.

## Highlights

- Added narrow Store SPI: `TaskCommandStore`, `TaskQueryStore`, and `TaskOperationsStore`, while keeping `TaskStore` as a compatibility facade.
- Added context-aware `TaskPayloadCodec` with serializer adapters so existing `TaskPayloadSerializer` beans remain compatible.
- Added ordered `TaskInterceptor` chain wiring through Spring beans, with default trace cleanup behavior preserved.
- Added `TaskTraceIdGenerator` with default `TraceContext` reuse and `rt-` prefixed fallback IDs.
- Added `TaskNameResolver` and `TaskHandlerMetadata` while preserving handler annotation consistency and duplicate task type detection.
- Split Admin REST auto-configuration into `reliable-task-admin-spring-boot-starter`; the base Spring Boot starter is now Worker-only.
- Added Flyway and Liquibase initial schema entry points and H2 MySQL-mode smoke tests.

## What's Included

- Store dependency narrowing across executor, template, retry, recovery, Admin, metrics, alert, and heartbeat paths.
- Source-compatible `TaskSerializer` deprecation guidance and `TaskPayloadCodec` customization path.
- Spring auto-configuration for custom `TaskPayloadCodec`, `TaskTraceIdGenerator`, `TaskNameResolver`, and ordered `TaskInterceptor` beans.
- Worker-only starter without Admin/Web dependencies by default.
- New Admin starter that explicitly opts applications into Admin REST auto-configuration and Web dependencies.
- Demo POM updated to depend on the Admin starter explicitly.
- Schema initialization options:
  - `db/schema.sql`
  - `db/migration/V1__init_reliable_task_schema.sql`
  - `db/changelog/db.changelog-master.yaml`

## Compatibility Notes

- ReliableTask remains in the `0.x` preview stage. APIs, configuration, and database schema may still change before `v1.0.0`.
- Existing `TaskStore` users keep source compatibility; new code should depend on the narrowest Store interface it needs.
- Existing `TaskPayloadSerializer` custom beans keep working. New context-aware payload customization should use `TaskPayloadCodec`.
- Existing handlers do not need to change. `@TaskHandler` and `getTaskType()` mismatch still fails registration.
- Applications that need Admin REST APIs must now add `reliable-task-admin-spring-boot-starter` explicitly in addition to enabling `reliable-task.admin.enabled=true`.
- Do not run `schema.sql`, Flyway, and Liquibase initializations together against the same database; they create the same ReliableTask tables and indexes.
- The retry-table task-id index is named `idx_retry_task_id` to keep initial schema validation portable while preserving indexed columns.
- ReliableTask is not published to Maven Central yet. Use a source build, local Maven installation, or a private Maven repository.

## Getting Started

Worker-only applications depend on the base starter:

```xml
<dependency>
    <groupId>com.reliabletask</groupId>
    <artifactId>reliable-task-spring-boot-starter</artifactId>
    <version>0.6.0</version>
</dependency>
```

Applications that need Admin REST APIs add the Admin starter explicitly:

```xml
<dependency>
    <groupId>com.reliabletask</groupId>
    <artifactId>reliable-task-admin-spring-boot-starter</artifactId>
    <version>0.6.0</version>
</dependency>
```

See the README for MySQL schema initialization, minimal configuration, Worker/Admin starter selection, extension-point migration guidance, Admin safety guidance, and demo instructions.
